## How to read this repository

- Start with the PRD to understand the product goal and MVP scope.
- Move to requirements and the functional inventory to understand what must be built.
- Read the data and ML sections before implementing the model.
- Use architecture, API, and Power BI docs when building the system and dashboard.
- Keep risks, validation, and Definition of Done aligned with the MVP.

## Priority convention

| Priority | Meaning |
|---|---|
| **P0 - Must have** | Required for the MVP to deliver value and be demonstrable |
| **P1 - Should have** | Adds meaningful value but does not block the first demo |
| **P2 - Could have** | Later evolution, experimental integration, or advanced automation |
| **Out of scope** | Not implemented in this version or not supported by evidence |
